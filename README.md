# Music Catalog API

REST API для управління музичним каталогом (артисти та треки) з JWT-автентифікацією.

## Технології
* **Node.js** та **Express**
* **PostgreSQL** (через **Prisma ORM**)
* **JWT** (JSON Web Tokens) та **bcryptjs** для безпеки
* **Jest** та **Supertest** для тестування (Unit та Integration)

## Передумови
Для запуску проєкту на вашому комп'ютері мають бути встановлені:
* Node.js
* PostgreSQL

## Інструкція із запуску

### 1. Встановлення залежностей
Склонуйте репозиторій та встановіть необхідні пакети:

```bash
# Клонування репозиторію
git clone https://github.com/lestaplus/music-catalog-api.git

# Встановлення залежностей
npm install
```

### 2. Налаштування середовища
Скопіюйте файл прикладу:

```bash
cp .env.example .env
```

Вміст файлу `.env.example`:
```env
DATABASE_URL="postgresql://postgres:your_password@localhost:5432/music_catalog?schema=public"
JWT_SECRET="secret-key"
```

### 3. Налаштування бази даних
Використайте вбудований скрипт для застосування міграцій та створення таблиць:

```bash
npm run migrate
```

*(Після міграції клієнт Prisma згенерується автоматично, але за потреби можна виконати оновлення вручну командою `npm run generate`).*

### 4. Запуск сервера
Для зручної розробки (з автоматичним перезавантаженням при змінах у коді):

```bash
npm run dev
```

Сервер запуститься на `http://localhost:3000`.

## Доступні NPM Скрипти
У проєкті налаштовані зручні команди. Виконуйте їх через `npm run <назва_скрипта>`:

* `start` — Запускає проєкт у звичайному режимі (`node src/index.js`).
* `dev` — Запускає проєкт у режимі розробки з `nodemon`.
* `test` — Запускає всі автотести через Jest (з підтримкою ES Modules).
* `migrate` — Застосовує міграції Prisma до бази даних.
* `studio` — Запускає зручний веб-інтерфейс бази даних (Prisma Studio).
* `generate` — Оновлює `Prisma Client`.

## Тестування
Проєкт використовує **Jest** для інтеграційного та модульного тестування.

```bash
npm run test
```
